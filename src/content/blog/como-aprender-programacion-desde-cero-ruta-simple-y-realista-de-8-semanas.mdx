Aprender programación desde cero no es “volverte senior en 2 meses”. Es algo mucho más útil (y alcanzable): **crear una base sólida** y llegar a construir **proyectos pequeños, reales y publicables**, con hábitos que puedas sostener.

En esta guía tienes una **ruta de 8 semanas**, diseñada para gente ocupada, con enfoque práctico y sin humo. Incluye: qué estudiar, qué practicar, proyectos por semana, errores típicos, dudas frecuentes y cómo convertir tu aprendizaje en resultados (portafolio, empleabilidad o productos).

## Qué puedes lograr en 8 semanas (siendo realistas)

Si cumples el plan con constancia, al finalizar podrás:

-   Entender los conceptos esenciales (variables, funciones, condicionales, ciclos, estructuras de datos básicas).
-   Leer documentación sin sentirte perdido.
-   Construir **2 a 4 proyectos simples** que puedas mostrar.
-   Subir código a GitHub y desplegar un proyecto web básico.
-   Tener claridad sobre tu siguiente paso (ruta web, datos, móvil, etc.).

Lo que **no** promete esta ruta: “ganar sueldos altos sin práctica” o “dominar todo”. La programación se aprende por capas.

## Antes de empezar: elige una ruta (sin bloquearte)

Para **aprender programación desde cero**, necesitas **un camino principal**. Elige una de estas dos rutas:

### Opción A (recomendada): Ruta Web con JavaScript

Ideal si te motiva ver resultados rápido (pantallas, botones, páginas).

**Aprenderás:** HTML + CSS + JavaScript + fundamentos de backend (básico) + despliegue.

### Opción B: Ruta Python (automatización / datos)

Ideal si te interesa lógica, scripts, análisis, automatizar tareas.

**Aprenderás:** Python + estructuras de datos + archivos + APIs + un mini proyecto.

**En esta guía usamos la Ruta Web con JavaScript** por ser la más visual y motivante para principiantes.

-   **Tiempo mínimo recomendado:** 60–90 min/día, 5 días por semana.
-   **Método:** 20% teoría / 80% práctica.
-   **Principio clave:** “Aprendo lo que uso en un proyecto”.

### Tu checklist de herramientas (Semana 1)

-   Un editor de código (VS Code).
-   Navegador (Chrome/Edge/Firefox).
-   Git + cuenta en GitHub.
-   Una carpeta de proyectos (orden desde el día 1).

## Ruta de 8 semanas para aprender programación desde cero (paso a paso)

### Semana 1: Fundamentos + configuración “pro”

**Objetivo:** preparar tu entorno y entender la base de cómo piensan los programas.

**Aprende:**

-   Cómo funciona un programa (entrada → proceso → salida).
-   Variables, tipos de datos, operadores.
-   Terminal básica (cd, ls/dir, mkdir).
-   Git básico: init, add, commit.

**Práctica (mini ejercicios):**

-   Calculadora simple (suma, resta, porcentaje).
-   Conversor: soles ↔ dólares (o unidades, lo que te motive).

**Proyecto de la semana (micro):** “Mi primera página personal” (HTML básico) con enlace a tu GitHub.

### Semana 2: HTML + CSS para construir interfaces simples

**Objetivo:** crear páginas que se vean ordenadas y claras.

**Aprende:**

-   HTML semántico (header, main, section, footer).
-   CSS: box model, flexbox, tipografías, espaciado.
-   Diseño simple: jerarquía visual (títulos, subtítulos, CTA).

**Práctica:**

-   Replica 2 secciones de una landing (hero + beneficios) con tu propio texto.

**Proyecto:** Landing de un servicio ficticio con:

-   Sección principal (beneficio + CTA)
-   3 ventajas
-   Testimonio (ficticio, pero con formato real)
-   Footer con contacto

### Semana 3: JavaScript básico (la lógica)

**Objetivo:** pensar en lógica sin frustrarte.

**Aprende:**

-   Condicionales (if/else)
-   Ciclos (for/while)
-   Funciones
-   Arrays (lo esencial)
-   Objetos (lo esencial)

**Práctica:**

-   Validador de formulario simple (nombre obligatorio, email con “@”, etc.).
-   Lista de tareas en consola (agregar, listar, eliminar).

**Proyecto:** Página con un formulario que muestre mensajes de validación con JS.

### Semana 4: JavaScript en el navegador (DOM + eventos)

**Objetivo:** crear interacción real (clicks, inputs, cambios en pantalla).

**Aprende:**

-   DOM: querySelector, createElement, classList
-   Eventos: click, submit, input
-   Buen hábito: separar funciones (no todo en una sola)

**Proyecto principal: To-Do List**

-   Agregar tarea
-   Marcar como completada
-   Eliminar tarea
-   Contador (pendientes / completadas)

**Extra recomendado:** Guardar tareas en LocalStorage.

### Semana 5: APIs + asincronía (cosas del mundo real)

**Objetivo:** consumir datos reales y entender async/await sin pánico.

**Aprende:**

-   fetch
-   Promesas y async/await
-   Manejo de errores (try/catch)

**Proyecto:** “Buscador con API” (elige uno):

-   Clima por ciudad
-   Películas por nombre
-   Tipo de cambio (si encuentras una API pública confiable)

**Enfoque People-First:** que el proyecto resuelva algo útil para ti o tu entorno.

### Semana 6: Introducción al backend (Node.js + Express)

**Objetivo:** entender cómo se crean endpoints y cómo viajan datos.

**Aprende:**

-   Qué es un servidor
-   Rutas (GET/POST)
-   JSON
-   Postman o Thunder Client (para probar endpoints)

**Proyecto:** API simple

-   Endpoint GET /items
-   Endpoint POST /items
-   Endpoint DELETE /items/:id

No necesitas “microservicios”. Solo lo esencial para entender el flujo.

### Semana 7: Base de datos (CRUD real)

**Objetivo:** persistir datos (que no se borren al reiniciar).

**Aprende:**

-   Concepto de tablas (o colecciones)
-   CRUD (Create, Read, Update, Delete)
-   Validaciones mínimas

**Proyecto:** mini sistema CRUD (elige uno):

-   Control de clientes
-   Registro de ventas simple
-   Inventario básico

**Meta:** que puedas decir “ya hice un CRUD con base de datos”.

### Semana 8: Despliegue + portafolio + siguiente paso

**Objetivo:** que tu trabajo sea visible y presentable.

**Aprende:**

-   Deploy de frontend (GitHub Pages / Vercel)
-   Deploy básico de backend (Render u otro)
-   README profesional (qué hace, cómo instalar, demo, screenshots)

**Entrega final (portafolio mínimo viable):**

-   1 landing (semana 2)
-   1 app JS (To-Do)
-   1 app con API
-   1 CRUD con BD (simple)

## Cómo saber si vas bien (sin compararte con otros)

Vas bien si:

-   Puedes explicar tu proyecto en 5–7 líneas.
-   Puedes arreglar un bug leyendo el error.
-   Tu código está en GitHub con commits.
-   Construyes algo cada semana, aunque sea pequeño.

Te estás desviando si:

-   Saltas de curso en curso sin terminar proyectos.
-   Te obsesionas con frameworks antes de dominar JS básico.
-   Solo miras videos y no practicas.
-   Buscas “la ruta perfecta” en lugar de ejecutar.

## Errores comunes al aprender programación desde cero

-   **Querer aprender 3 lenguajes a la vez:** elige uno y quédate.
-   **Ignorar Git:** luego te frena al buscar prácticas o trabajo.
-   **Copiar/pegar sin entender:** copia, pero reescribe y modifica.
-   **No hacer proyectos:** la teoría sin práctica se evapora.
-   **Perseguir “la ruta perfecta”:** no existe; la constancia sí.

## Preguntas frecuentes sobre aprender programación desde cero

### ¿Necesito saber matemáticas avanzadas?

No para empezar. Necesitas **lógica básica** y paciencia. Matemática avanzada dependerá del área (IA, gráficos, etc.).

### ¿Cuánto debo practicar para ver resultados?

Con **5–7 horas semanales** bien enfocadas ya verás avances. Con 10 horas, mejor. Más importante que la cantidad: **continuidad**.

### ¿Qué curso debo comprar?

Si recién empiezas, prioriza uno que:

-   Te obligue a hacer proyectos
-   Tenga ejercicios
-   Te enseñe a leer documentación

Si un curso solo “explica”, te va a costar mucho más.

### ¿Cuándo aprendo frameworks (React, etc.)?

Cuando ya puedas:

-   Manipular el DOM con JS
-   Consumir APIs
-   Estructurar funciones sin caos

Normalmente después de esta ruta (8–12 semanas).
